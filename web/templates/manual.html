사용 매뉴얼

로그인 → 광고주 등록 → 보고서 생성 · 다운로드. 처음이면 이 순서대로 따라오세요.

!

지금은 “테스트 서버”입니다

기능을 확인해보는 임시 환경이라 실제 운영(라이브) 서버와 몇 가지 차이가 있습니다. 아래 테스트 vs 라이브 비교표를 꼭 확인하세요. 특히 보고서 생성 중에는 그 페이지를 벗어나지 마세요.

1

로그인

  • 관리자가 만들어 준 이메일 · 비밀번호로 로그인합니다.
  • 로그인하면 [광고주 · 생성] 화면으로 이동합니다.
2

광고주 등록

보고서를 뽑을 광고주를 먼저 등록합니다. 두 가지 방법이 있어요.

방법 A · 시트로 한 번에 (추천)

  1. [템플릿 받기] 눌러 엑셀 양식 다운로드
  2. 양식에 광고주별로 한 줄씩 입력 (아래 열 설명 참고)
  3. 파일 선택 → [업로드]

추가: 기존 유지+신규만 / 전체 교체: 내 광고주 전부 지우고 새로 채움

방법 B · 하나씩 직접 입력

[＋ 광고주 하나씩 추가]를 펼쳐서 이름·customer_id·키를 입력하고 저장합니다.

입력 항목 (네이버 검색광고에서 발급)

  • customer_id — 광고 계정 번호 (예: 4273)
  • api_key — 액세스 라이선스 (0100…으로 시작)
  • api_secret — 비밀키 (AQ…로 시작)

발급 위치: tool.searchad.naver.com → 도구 → API 사용 관리. 키가 없으면 목록에 “키 없음”으로 표시되고 생성이 안 됩니다.

· 목록 우측 검색창으로 광고주를 찾고, 행에 마우스를 올리면 나오는 🗑로 삭제할 수 있습니다.

3

보고서 생성

  • 상단에서 연도·월을 고릅니다 (기본은 지난달).
  • 광고주 한 곳만 만들려면 그 행의 [생성] 버튼.
  • 키가 등록된 광고주 전부를 만들려면 상단 [전체 생성] 버튼.
  • 상태 칸이 수집 중 → 엑셀 작성 → ✓ 완료로 바뀌면 다운로드 링크가 나옵니다.

⚠️ 테스트 서버 주의: 생성 버튼을 누른 뒤 “완료/실패”가 뜰 때까지 그 페이지에 그대로 있으세요. 중간에 다른 화면으로 나가면 생성이 멈출 수 있습니다. (라이브 서버에서는 나가도 완료됩니다 — 아래 표 참고)

4

생성된 보고서 · 다운로드

  • 왼쪽 [생성된 보고서]에서 만든 보고서를 확인합니다.
  • 전체 / 완료 / 실패 탭과 생성일 · 보고월 · 광고주명 필터로 찾을 수 있습니다.
  • 개별 다운로드, 또는 체크 후 선택 다운로드/전체 다운로드(zip)로 한 번에 받습니다.
  • 필요 없는 보고서는 선택 삭제 또는 행의 🗑로 지웁니다.
★

팀원 관리 (관리자만)

  • [팀원 관리]에서 팀장·팀원 계정을 만들어 줍니다.
  • 격리 원칙: 팀장/팀원은 자기가 등록한 광고주·보고서만 봅니다. 서로 안 보입니다.
  • 관리자는 전체를 볼 수 있습니다.

⚖️ 지금(테스트 서버) vs 앞으로(라이브 서버)

기능은 같지만 안정성·속도·데이터 보관에서 차이가 있습니다.

항목 지금 · 테스트 서버 앞으로 · 라이브 서버
생성 중 페이지 이탈 나가면 멈출 수 있음 → 생성 중엔 대기 나가도 끝까지 완료됨
첫 접속 속도 잠깐 느릴 수 있음(대기 후 깨어남) 항상 즉시
응답 속도(위치) 도쿄 서버 — 약간의 지연 서울 서버 — 더 빠름
데이터 보관 테스트용이라 초기화될 수 있음 안정적으로 보관·백업
동시 대량 생성 소량 확인용(한두 건씩 권장) 대량·야간 배치 대응
이메일 자동발송·스케줄 미제공(추후) 도입 예정

막히면?

계정 발급·API 키·오류 등 문제가 생기면 관리자에게 문의하세요. 이 화면은 언제든 왼쪽 [사용 매뉴얼]에서 다시 볼 수 있습니다.
