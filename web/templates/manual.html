사용 매뉴얼

처음이신가요? 이 도구는 네이버 파워링크 광고 보고서를 자동으로 만들어 주는 사내 시스템입니다. 아래 순서대로 따라오시면 됩니다.

!

지금은 “테스트 서버”입니다

실제 운영 전 미리 써보는 환경입니다. 기능은 동일하고, 속도·데이터 보관에서 약간의 차이만 있습니다(아래 비교표). 생성은 요청만 하면 서버가 알아서 처리하니 자유롭게 쓰세요.

먼저, 내 역할이 뭔가요?

관리자

전체 팀·광고주·보고서를 봄. 계정 생성·소속 지정·비번 재설정.

팀장

우리 팀(나 + 소속 팀원)의 광고주·보고서를 봄. 광고주 등록·생성, 팀원에게 보고서 할당.

팀원

내 광고주 + 팀장이 할당해 준 보고서를 봄. 등록·생성도 가능.

계정·역할은 관리자가 만들어 줍니다. 로그인 정보는 관리자에게 받으세요.

1

로그인

  • 관리자가 준 이메일·비밀번호로 로그인합니다.
  • 비밀번호를 잊었다면 관리자가 재설정해 줄 수 있습니다.
2

광고주 등록

보고서를 뽑을 광고주를 먼저 등록합니다. 방법은 두 가지.

A · 시트로 한 번에 (추천)

  1. [템플릿 받기] → 엑셀 양식 다운로드
  2. 광고주별로 한 줄씩 입력
  3. 파일 선택 → [업로드]

추가=기존 유지+신규만 / 전체 교체=내 광고주 전부 지우고 새로 채움

B · 하나씩 직접 입력

[＋ 광고주 하나씩 추가]를 펼쳐 입력·저장.

등록 후 목록에서 연필(수정)로 언제든 고칠 수 있습니다(키 교체·오타).

입력 항목 (네이버 검색광고에서 발급)

  • customer_id — 광고 계정 번호 (예: 4273)
  • api_key — 액세스 라이선스 (0100…으로 시작)
  • api_secret — 비밀키 (AQ…로 시작)

발급: tool.searchad.naver.com → 도구 → API 사용 관리

✓ 키 검증 자동

등록하면 키가 맞는지 1회 확인합니다. 틀리면 목록에 ‘정보 없음’으로 표시 → 연필로 고치세요.

🔎 찾기 편하게

검색창, 키 상태·담당 팀장 필터, 열 제목 클릭 정렬, KPI 카드 클릭 필터를 쓰세요.

⚠️ 삭제 주의: 광고주를 삭제하면 그 광고주의 보고서도 함께 삭제됩니다. 팀원에게 할당했던 보고서면 해당 팀원에게 삭제 알림이 갑니다. 삭제 전 확인창의 안내를 꼭 읽어보세요.

3

보고서 생성

  • 상단에서 연도·월을 고릅니다(기본 지난달).
  • 한 곳만: 그 행의 [생성] · 전부: 상단 [전체 생성](키 있는 광고주 한 번에 예약).
  • 같은 광고주·같은 달을 다시 만들면 새로 쌓이지 않고 최신본으로 덮어씁니다.

요청하고 나가도 됩니다 👍 서버가 백그라운드로 처리하므로 창을 닫거나 로그아웃해도 [생성된 보고서]에 예약 → 생성 중 → 완료로 하나씩 나타납니다(자동 새로고침).

4

생성된 보고서 · 다운로드 · 할당

  • [생성된 보고서]에서 전체/진행 중/완료/실패 탭 + 생성일·보고월·담당 팀원 필터로 찾기.
  • 개별 다운로드, 체크 후 선택/전체 다운로드(zip)로 한 번에.
  • 실패한 건 [다시 생성](개별·실패 전체)으로 재시도. 키 오류면 [키 고치기]로 이동.
  • 팀원에게 할당(팀장): 완료 보고서 체크 → [팀원에게 할당] → 팀원 선택. 취소는 ‘→OOO 할당’ 옆 ✕.
5

팀원이 받기 · 알림 🔔

  • 팀원은 로그인하면 할당받은 보고서가 [생성된 보고서]에 보입니다(본인 것 + 할당분).
  • 왼쪽 아래 🔔 종 아이콘에 새 알림이 뜹니다: 할당됨 / 업데이트됨(재생성) / 삭제됨.
★

관리자 — 팀원 관리

  • [팀원 관리]에서 계정 생성(역할: 팀장/팀원), 소속 팀장 드롭다운 지정.
  • 비밀번호 재설정, 계정 삭제 가능.

⚠️ 계정 삭제 주의: 팀장 계정을 삭제하면 그 팀장이 등록한 광고주와 보고서가 모두 함께 삭제됩니다(되돌릴 수 없음). 소속 팀원 계정은 남지만 ‘소속’이 해제됩니다. 할당됐던 보고서는 해당 팀원에게 삭제 알림이 갑니다.

❓ 자주 막히는 것

‘키 없음’아직 API 키를 안 넣음 → 연필(수정)로 키 입력 후 생성.
‘정보 없음’키가 틀렸거나 만료 → 연필로 올바른 키로 교체(자동 재검증).
생성 ‘실패’사유가 한국어로 표시됨. 키 오류면 [키 고치기], 지연이면 [다시 생성].
업로드 일부 누락업로드 후 ‘결과 패널’에서 건너뛴 행·사유 확인(주로 customer_id 빈칸).

⚖️ 지금(테스트) vs 앞으로(라이브)

항목테스트 서버라이브 서버
생성 중 이탈나가도 됨나가도 됨
첫 접속 속도잠깐 느릴 수 있음항상 즉시
응답 속도도쿄 서버서울 서버(더 빠름)
데이터 보관초기화될 수 있음안정적 보관·백업
이메일 자동발송미제공(추후)도입 예정
