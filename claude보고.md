1. 지금까지 완성된 것 (보고서B, 파워링크)
검증 완료 (에듀윌 실계정, LibreOffice 재계산 대조):

영역	상태
RAW 2종(파워링크·소재)	실데이터 채움, 수식 참조 원천 ✅
Summary (일별·주차·합계)	RAW→수식 자동계산, 1일~말일 전부 ✅
상세데이터(기기·캠페인·그룹)	RAW직접 SUMIFS로 정합, TOTAL=월합 ✅
소재별_주간 / 키워드(평일·주말)	값+수식, 키워드 합계=기간 총합 ✅
AI 코멘트	실데이터와 수치 일치, 검수 대상 ✅
안정성	수집 재시도, 서식·병합·숨김행 처리 ✅
핵심 제약(이미 해결): 기기=캠페인명 _PC/_MO(API 7일 제한), 브랜드검색 자동 제외, 키워드 20만개→상위50+기타행, 수집 rate-limit 대응.

한 줄 요약: "customer_id + 연/월만 주면 검증 가능한 보고서B 1건이 나온다" 는 상태. 단건은 완성. 지금부터는 이걸 수백 건 돌리는 파이프라인이 과제입니다.

2. 큰 그림 — 6단계 파이프라인

① 온보딩        ② 수집           ③ 생성          ④ 자동 QA        ⑤ 검수·승인      ⑥ 발송·기록
(크리덴셜/광고주) → (네이버 API) → (보고서B.xlsx) → (정합성 게이트) → (마케터, 사람) → (이메일 자동)
   담당자          시스템          시스템          시스템 ★          마케터           시스템
★ 핵심 인사이트: 대량에서는 "사람이 100건을 눈으로 검수"가 불가능합니다. 그래서 우리가 만든 정합성 검증(RAW=Summary, 키워드합=월합, 누락일 0, 기기 정합)을 ④ 자동 게이트로 만들어, 통과한 것만 사람에게 올립니다. 마케터는 "숫자 검증"이 아니라 "코멘트/인사이트 검수"에만 집중. 이게 규모의 핵심입니다.

3. 역할 분담
역할	담당	책임

운영/아키텍트	본인	파이프라인 설계·운영, 월배치 스케줄, QA 게이트 룰 관리, 장애 대응, 매체별 템플릿 관리
마케터	광고주별	QA 통과분의 코멘트·인사이트 검수/수정 → 승인. 광고주 커뮤니케이션
시스템	자동	수집·시트생성·정합성검증·코멘트초안·발송·로그
4. 단계별 상세 + 개선 아이디어
① 온보딩 (크리덴셜 인입) — 예시(엑셀 정리)를 이렇게 발전:

담당자가 편한 스프레드시트(엑셀/구글시트)를 "원본" 으로 유지: 광고주명 | 파워링크 customer_id | api_key | api_secret | 매체 | 담당마케터 | 수신이메일 | 활성
시스템은 이걸 업로드/동기화로 가져와 검증 후 저장. ⚠️ accounts.json 직접 편집 금지 — 키는 암호화 저장, 접근권한 분리(온보딩 담당자만 입력, 로그에 키 노출 금지).
더 나은 안: 구글시트를 SoT로 두고 자동 동기화(담당자는 시트만 관리, 시스템이 주기적 반영) → 온보딩 마찰 최소.
② 수집 — 규모 대응 핵심:

광고주마다 다른 API키 = 다른 rate-limit 버킷 → 광고주 단위 병렬이 안전(계정 내부는 동시성 4 유지). 100개 광고주 × 5 병렬 ≈ 1.5~2시간.
월배치 스케줄(예: 매월 1일 새벽) — 야간에 돌려 아침엔 검수만.
멱등성: 같은 광고주·월 재생성해도 안전(현재 구조 OK). 실패분만 재시도.
③ 생성 — 이미 완성. 광고주별 매체 구성에 따라 템플릿 매핑(현재 파워링크; 향후 카카오/구글 시트도 같은 방식 확장 가능).

④ 자동 QA 게이트 (신규, 가장 중요) — 우리가 만든 검증을 코드 게이트로:

통과 조건: Summary합계=RAW총합 · 키워드 평일+주말=월합 · 일자 1~말일 누락 0 · 기기 PC+모바일+기타=월합 · 소재 VLOOKUP 실패 0.
실패 시 → "재생성 필요"로 자동 분류, 마케터에게 안 올림. 원인(누락일/키불일치) 로그.
효과: 마케터는 "숫자는 이미 맞다" 전제하에 코멘트만 본다.
⑤ 검수·승인 (사람) — 로그인 UI의 진짜 용도:

회원로그인은 "혼자 1건 뽑기"가 아니라 "이달 내 담당 광고주 20건 검수 대시보드" 로 써야 의미 있음.
화면: 광고주 리스트(상태: 생성완료/검수대기/승인/발송), 코멘트 인라인 수정, 원클릭 승인, 미리보기.
더 나은 안: 전월 대비 diff를 코멘트 위에 표시 → 검수 속도↑. AI 코멘트는 초안(draft), 승인 없이는 발송 금지(정확성·책임 이슈).
⑥ 발송·기록 —

승인 즉시 템플릿 이메일 + 첨부(xlsx) 자동 발송, 발송 로그/수신확인 기록.
예약발송(예: 매월 5일 일괄), 실패 시 재발송. 감사추적(누가 언제 승인·발송).
5. 규모 확장 시 리스크 & 대응
리스크	대응

코멘트 API 비용	배치·캐시, 초안만 생성
광고주별 특수사항(확장검색·미태깅 기기)	템플릿별 주석으로 마케터에게 명시
6. 추천 구축 순서 (MVP → 확장)
MVP: 스프레드시트 인입 → 배치 생성 → QA 게이트 → 검수 대시보드 → 수동발송. (핵심 뼈대)
자동화: 월 스케줄 배치 + 승인 후 자동 이메일 발송.
효율화: 전월 대비 diff, 코멘트 수정률 추적→프롬프트 개선, 구글시트 SoT 동기화.
확장: 매체 추가(카카오/구글), 광고주 셀프 리포트 포털.
미팅용 한 문장: "단건 보고서B는 검증까지 완성됐고, 다음은 '스프레드시트로 광고주 등록 → 야간 자동 생성 → 정합성 자동검증 통과분만 마케터가 코멘트 검수·승인 → 자동 발송' 파이프라인. 사람은 숫자가 아니라 인사이트만 본다가 핵심."
