④ 자동 정합성 게이트가 핵심 — Summary=RAW, 키워드합=월합, 누락일 0을 코드로 자동 판정. 실패분은 "재생성 필요"로 자동 분류돼 사람에게 안 올라감. 마케터는 인사이트만 검수.

03 · 역할

온보딩 담당자

API키·customer_id·수신 이메일을 스프레드시트에 정리 → 등록. 신규 광고주 추가.

운영 · 아키텍트 (본인)

파이프라인 설계·운영, 월배치 스케줄, QA 룰 관리, 장애 대응, 템플릿 관리.

마케터

QA 통과분의 코멘트·인사이트 검수/수정 → 승인. 광고주 커뮤니케이션.

시스템 (자동)

수집·시트생성·정합성검증·코멘트초안·발송·로그.

04 · 단계별 상세 + 개선 아이디어

🗂️

온보딩 — 크리덴셜 인입

담당자가 편한 스프레드시트를 원본으로(광고주·customer_id·키·담당·수신이메일). 시스템은 검증 후 저장.

⤷ 구글시트 자동 동기화 · ⚠️ 키 암호화·json 직접편집 금지

📡

수집 — 규모 대응

광고주마다 다른 API키 = 다른 rate-limit → 광고주 단위 병렬. 100개 × 5병렬 ≈ 1.5~2시간.

⤷ 월배치 야간 스케줄, 실패분만 재시도(멱등)

✅

자동 QA 게이트 — 가장 중요 (신규)

✓ Summary 합계 = RAW 총합
✓ 키워드 평일+주말 = 월합
✓ 일자 1~말일 누락 0
✓ 기기 PC+모바일+기타 = 월합
✓ 소재 VLOOKUP 실패 0
✓ 코멘트 수치 = 실데이터

⤷ 실패 → 재생성 필요 자동분류, 마케터에게 안 올림

✍️

검수·승인 — 로그인 UI의 진짜 용도

"이달 내 담당 광고주 N건 검수 대시보드". 상태·코멘트 인라인 수정·원클릭 승인·미리보기.

⤷ 전월 대비 diff 표시 · AI 코멘트는 초안, 승인 전 발송 금지

🚀

발송·기록

승인 즉시 템플릿 이메일 + 첨부 자동 발송, 발송 로그·수신확인, 감사추적(누가·언제).

05 · 리스크 & 대응

네이버 rate-limit

광고주 병렬 + 동시성 제한 + 재시도 + 야간 배치

AI 코멘트 오류·책임

승인 없이 발송 금지 — 사람 검수 필수

크리덴셜 유출

키 암호화 · 접근분리 · 로그 마스킹

데이터 누락 은폐

자동 QA 게이트가 발송 전 차단

광고주별 특수사항

확장검색·미태깅 기기 → 템플릿 주석으로 명시

06 · 구축 로드맵

PHASE 1

MVP · 뼈대

인입→생성→QA→검수→수동발송

PHASE 2

자동화

월 스케줄 + 승인 후 자동 발송

PHASE 3

효율화

전월 diff · 프롬프트 개선 · 시트 동기화

PHASE 4

확장

매체 추가(카카오·구글) · 셀프 포털

미팅용 한 문장

"단건 보고서B는 검증까지 완성됐고, 다음은 스프레드시트로 광고주 등록 → 야간 자동 생성 → 정합성 자동검증 통과분만 마케터가 검수·승인 → 자동 발송 파이프라인. 사람은 숫자가 아니라 인사이트만 본다가 핵심."
